# 飞任务 - 任务管理小程序设计思考

## 产品灵魂与意象锚点

**核心意象**：清晨的办公桌，一杯刚冲好的咖啡旁，整齐排列的便签卡片。每张便签代表一个任务，颜色标记着紧急程度，折角显示着完成状态。这不是复杂的系统，而是简单、可靠、随手可用的"数字便签"。

**气质关键词**：
- **高效**：不拖泥带水，直指目标
- **清晰**：信息一目了然，层级分明
- **可靠**：不遗漏任何任务，准时提醒
- **协作**：团队间的信任传递，责任清晰

## 设计风格定位

**整体风格**：工具型效率美学 - 借鉴 Notion、Linear、Things 3 的克制设计语言
- **简洁克制**：无多余装饰，每个元素都有明确功能
- **信息优先**：内容为王，UI 是内容的载体
- **微交互精致**：状态切换、卡片展开等细节打磨到位
- **深色模式友好**：白天清晰、夜间舒适的双模式

## 配色方案

### 主色调 - 效率蓝（Productivity Blue）
- **灵感来源**：天空清晨的蓝色，代表着新一天的开始和无限可能
- **主色**：`#1890FF` - 用于主要操作、状态标识、强调元素
- **主色深色**：`#0066CC` - 用于深色模式下的主色，降低饱和度避免刺眼

### 辅助色 - 任务状态色系
- **待办（Todo）**：`#FAAD14` 橙黄 - 温暖提醒，等待开始
- **进行中（InProgress）**：`#52C41A` 绿色 - 生机勃勃，正在推进
- **已完成（Done）**：`#8C8C8C` 灰色 - 沉静归档，已完成使命
- **紧急（Urgent）**：`#F5222D` 红色 - 紧急警示，需要立即关注

### 中性色 - 信息层级色系
- **深色模式**：
  - 背景：`#1F1F1F`（主背景）`#2A2A2A`（卡片背景）
  - 文字：`#FFFFFF`（主要）`#BFBFBF`（次要）`#8C8C8C`（辅助）
  - 边框：`#3A3A3A`
- **浅色模式**：
  - 背景：`#FFFFFF`（主背景）`#F5F5F5`（卡片背景）
  - 文字：`#262626`（主要）`#595959`（次要）`#8C8C8C`（辅助）
  - 边框：`#E8E8E8`

## 字体排版

**字体选择**：
- **中文**：思源黑体（Noto Sans SC）- 简洁现代，适合工具型应用
- **数字/英文**：Inter - Google 出品的效率型字体，数字清晰易读

**排版节奏**：
- **页面标题**：20px / 粗体 / 主文字色 - 清晰标识当前位置
- **任务标题**：16px / 中等粗细 / 主文字色 - 任务核心信息
- **次要信息**：14px / 正常 / 次文字色 - 负责人、时间等辅助信息
- **辅助说明**：12px / 正常 / 辅文字色 - 描述、评论等补充内容
- **数字标识**：Inter 16px / 粗体 / 主色 - 任务数量、状态统计

## 视觉策略

### 任务卡片设计
- **卡片形态**：圆角 12px，轻微阴影，白色/深灰背景
- **信息层级**：
  1. 标题（最大最显眼）
  2. 状态标签（颜色区分）
  3. 负责人头像+名字
  4. 截止时间（倒计时提示）
- **交互反馈**：hover 时轻微上浮，点击时卡片放大进入详情

### 状态标识设计
- **状态标签**：圆角胶囊形，左侧带小圆点色标
- **紧急标识**：任务卡片左侧红色边条，视觉上立即抓住注意力
- **完成状态**：任务标题前带勾选图标，视觉上区分已完成任务

### 列表布局
- **分页切换**：顶部 Tabs（我创建的/我负责的/全部），不占过多空间
- **卡片间距**：12px 垂直间距，视觉上卡片独立但整体连贯
- **空状态**：居中显示空状态图标+提示文案，引导用户创建任务

## 动效与交互

### 状态切换动效
- **完成动画**：勾选图标从无到有渐现，背景色渐变到灰色，标题文字淡化
- **创建动画**：新任务卡片从底部滑入，短暂放大后恢复正常大小
- **删除动画**：卡片轻微抖动后渐隐消失

### 列表滚动
- **平滑滚动**：使用原生 scroll-view，带轻微惯性滚动
- **下拉刷新**：顶部下拉触发刷新，带刷新图标动画

### 页面切换
- **卡片点击**：卡片轻微放大后进入详情页
- **返回动画**：详情页渐隐，列表页渐现

## 页面结构

### 首页（任务列表）
- **顶部区域**：
  - 应用标题"飞任务"
  - 状态筛选 Tabs（我创建的/我负责的/全部）
  - 快速创建按钮（悬浮在右上角）
- **主体区域**：
  - 任务卡片列表（可滚动）
  - 空状态提示（无任务时）
- **底部区域**：
  - TabBar（任务/创建/我的）

### 任务创建页
- **表单区域**：
  - 任务标题输入（大字号，突出）
  - 负责人选择（头像+名字列表）
  - 截止时间选择（日历组件）
  - 任务描述（多行文本框）
- **语音创建**：
  - 录音按钮（悬浮在输入框旁）
  - 录音状态指示（录音中动画）
- **操作按钮**：
  - 取消/创建按钮（底部固定）

### 任务详情页
- **头部信息**：
  - 任务标题（大字号）
  - 状态标签（可切换）
  - 创建人、负责人、时间信息
- **描述区域**：
  - 任务描述内容
  - 编辑按钮
- **评论区域**：
  - 评论列表（头像+内容+时间）
  - 评论输入框（底部固定）
- **历史记录**：
  - 时间线形式展示状态变更历史
- **操作按钮**：
  - 状态切换、删除等操作

### 个人中心页
- **用户信息**：
  - 头像、名字
  - 登录/注册入口
- **设置区域**：
  - 深色模式切换
  - 通知设置
  - 其他偏好设置
- **底部区域**：
  - 版本信息、退出登录

## 设计禁忌

**严格禁止**：
- ❌ 不要使用复杂的渐变背景 - 简洁是效率的核心
- ❌ 不要使用大尺寸装饰性图片 - 工具型应用不需要视觉冲击
- ❌ 不要使用过多的动画效果 - 只在必要交互时使用微动效
- ❌ 不要使用 emoji 或表情图标 - 专业感是效率应用的基础
- ❌ 不要使用饱和度过高的颜色 - 长时间使用会视觉疲劳
- ❌ 不要将紧急任务红色用于其他元素 - 保持语义色的专一性
- ❌ 不要在深色模式下使用纯白文字 - 降低到灰白色更舒适

## 深色模式适配原则

**核心原则**：
- 降低主色饱和度，避免夜间刺眼
- 增加背景层次，避免纯黑单调
- 文字使用灰白色而非纯白
- 边框和分割线使用深灰色而非黑色
- 卡片使用深灰而非纯黑，保持层次感

## 组件使用原则

**UI 组件优先**：
- 按钮、输入框、卡片、标签、弹窗、Toast、Tabs、Dialog 等通用组件，必须优先使用 `@/components/ui/*`
- 不在页面中用 View/Text 手搓这些通用 UI
- 页面实现前先拆分 UI 单元，再把每个单元映射到组件库

**页面组件选型**：
- 任务列表卡片 → `Card` + `Badge`
- 状态筛选 → `Tabs`
- 任务创建表单 → `Input` + `Textarea` + `Calendar` + `Button`
- 负责人选择 → `Avatar` + `Checkbox` 或自定义选择器
- 评论输入 → `Textarea` + `Button`
- 状态切换 → `Select` 或自定义状态选择器
- 空状态提示 → 自定义空状态组件或 `Alert`
- 加载态 → `Skeleton`
- 弹窗确认 → `AlertDialog` 或 `Dialog`
- 提示反馈 → `Toast` 或 `Sonner`

## 导航结构

**TabBar 配置**：
- **任务**（首页）：任务列表，默认页
- **创建**：快速创建任务页
- **我的**：个人中心页

**页面跳转**：
- TabBar 页面使用 `switchTab()`
- 任务详情页使用 `navigateTo()`
- 创建完成后返回列表页使用 `navigateBack()`
