## 修复：WXSS 编译错误（真机调试报错）

**问题**：真机调试时微信开发者工具报 WXSS 编译错误 `error at token '['`，原因是 Tailwind CSS v4 生成的 CSS 包含 WXSS 不兼容语法。

**不兼容模式**：
- `@supports (color: color(display-p3 0 0 0%)){...}` 块
- `[data-orientation=vertical]` 等未加引号的属性选择器值
- `[class*=size-]` 通配符属性选择器

**尝试1（失败）**：PostCSS 插件 `config/postcss-wxss-compat.ts` — 注册到 Vite `config.css.postcss.plugins`，但 Taro 小程序 CSS 管道不走此通路，插件未生效。

**尝试2（失败）**：Vite `writeBundle` 钩子插件 `config/vite-plugin-wxss-cleanup.ts`
- 在构建完成后处理 `dist/*.wxss` 文件
- 但 Taro 的 Vite 封装未触发 `writeBundle` 钩子，且 `@supports` 块含嵌套花括号，简单正则无法正确匹配

**最终方案（成功）**：独立 Node.js 后处理脚本 `scripts/clean-wxss.js`
- 使用**逐字符解析 + 花括号深度计数**，正确处理嵌套 `@supports` 块
- `removeSupportsBlocks()`: 查找 `@supports` 后从 `{` 开始计数，depth 归零即整块移除
- `removeHasPseudo()`: 同样括号深度计数移除 `:has(...)` 伪类
- `quoteAttrValues()`: `[data-xxx=yyy]` → `[data-xxx="yyy"]`
- `simplifyWildcardAttrs()`: `:not([class*=xxx])` → `:not([class])`
- 修改 `package.json` 的 `build:weapp`/`build:tt` 命令，在 taro build 后用 `&&` 链式执行
- 从 `config/index.ts` 移除无效的 Vite 插件注册

**验证结果**：7项 WXSS 不兼容检查全部清零，花括号完全平衡（495:495）

## 修复：创建任务后主页不显示（缺少 useDidShow 刷新）

**问题根因**：`src/pages/index/index.tsx` 的 `fetchTasks` 仅在 `useEffect` 中通过 `activeTab` 变化触发。微信小程序 tab 页面有缓存机制，从"创建"页 `switchTab` 回"任务"页时 `activeTab` 不变，`useEffect` 不重新执行，新创建的任务不显示。

**另一个问题**：请求失败时静默切换到 Mock 数据（"示例任务一/二"），掩盖了真实错误（如 Render 冷启动超时、域名白名单拦截等）。

**修复内容**：
1. **添加 `useDidShow` 刷新**：页面每次重新显示（tab 切换回来）时自动调用 `fetchTasks` 刷新列表
2. **`fetchTasks` 改为 `useCallback`**：依赖 `[currentUserId, activeTab]`，`useEffect` 依赖 `fetchTasks`
3. **移除 Mock 数据**：请求失败时显示真实错误信息 + `setTasks([])` 清空列表
4. **添加 `timeout: 15000`**：应对 Render 免费版冷启动慢的问题
5. **创建任务请求也加 `timeout: 15000`**，错误提示从 `showToast` 改为 `showModal` 显示详细信息

**验证**：后端 API 三种 filter（all/created/assigned）均正常返回数据，响应时间 1-2 秒。dist 构建成功，确认包含 `useDidShow` 逻辑。

## 修复：个人中心统计数据不自动更新

**问题根因**：`src/pages/profile/index.tsx` 的 `fetchUserInfo` 只在首次 `useEffect` 中调用一次。作为 tab 页面，微信会缓存页面实例，之后切换回来不会重新获取数据，导致"我创建的/我负责的/已完成"数字不更新。

**修复内容**：
1. 添加 `useDidShow`：每次切回"我的"页面时自动刷新用户信息和任务统计
2. `fetchUserInfo` 改为 `useCallback`，依赖 `[isLoggedIn, currentUserId]`
3. 请求添加 `timeout: 15000`（应对 Render 冷启动）
4. 获取任务列表时传入 `userId` 和 `filter: 'all'` 参数
5. 分离登录状态检查（`useEffect`）和数据获取（`useDidShow` + `useEffect`）
