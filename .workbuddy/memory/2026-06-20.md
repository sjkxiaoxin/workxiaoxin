## 修复：创建小队失败 Bug

**问题根因**：`src/pages/team/index.tsx` 第 32 行用户 ID 硬编码为 `'user_001'`（下划线），而项目其他地方（profile 登录写入 storage、API 调用）统一使用 `'user-001'`（连字符）。

**后端机制**：`teams` 和 `team_members` 表的 `creator_id`/`user_id` 字段有外键约束引用 `users.id`，若传入不存在的 ID（`user_001`），Supabase 插入时报外键违反错误，后端抛 `BadRequestException`，前端 catch 显示「创建失败」。

**修复方案**：将 `useState('user_001')` 改为 `useState(() => Taro.getStorageSync('userId') || 'user-001')`，从登录写入的 storage 读取真实用户 ID，与项目其他部分保持一致。

**文件**：`src/pages/team/index.tsx` 第 32 行

## 修复：WXSS 编译错误（真机调试报错）

**问题**：真机调试时微信开发者工具报 WXSS 编译错误 `error at token '['`，原因是 Tailwind CSS v4 生成的 CSS 包含 WXSS 不兼容语法。

**不兼容模式**：
- `@supports (color: color(display-p3 0 0 0%)){...}` 块
- `[data-orientation=vertical]` 等未加引号的属性选择器值
- `[class*=size-]` 通配符属性选择器

**尝试1（失败）**：PostCSS 插件 `config/postcss-wxss-compat.ts` — 注册到 Vite `config.css.postcss.plugins`，但 Taro 小程序 CSS 管道不走此通路，插件未生效。

**尝试2（失败）**：Vite `writeBundle` 钩子插件 `config/vite-plugin-wxss-cleanup.ts`
- 在构建完成后处理 `dist/*.wxss` 文件
- 但 Taro 的 Vite 封装未触发 `writeBundle` 钩子，且 `@supports` 块含嵌套花括号，简单正则无法正确匹配

**最终方案（成功）**：独立 Node.js 后处理脚本 `scripts/clean-wxss.js`
- 使用**逐字符解析 + 花括号深度计数**，正确处理嵌套 `@supports` 块
- `removeSupportsBlocks()`: 查找 `@supports` 后从 `{` 开始计数，depth 归零即整块移除
- `removeHasPseudo()`: 同样括号深度计数移除 `:has(...)` 伪类
- `quoteAttrValues()`: `[data-xxx=yyy]` → `[data-xxx="yyy"]`
- `simplifyWildcardAttrs()`: `:not([class*=xxx])` → `:not([class])`
- 修改 `package.json` 的 `build:weapp`/`build:tt` 命令，在 taro build 后用 `&&` 链式执行
- 从 `config/index.ts` 移除无效的 Vite 插件注册

**验证结果**：7项 WXSS 不兼容检查全部清零，花括号完全平衡（495:495）
