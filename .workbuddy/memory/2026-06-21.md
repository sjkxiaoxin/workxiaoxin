## 新增功能：统计数字点击跳转查看具体事项

**需求**：我的页面中"我创建的"、"我负责的"、"已完成"数字可以点击，跳转到任务列表页查看对应事项。

**实现方案**：
1. **profile页（`src/pages/profile/index.tsx`）**：三个统计数字 View 加 `onClick`，先 `Taro.switchTab` 跳转到 index，再用 `Taro.setStorageSync('pendingFilter', 'created'/'assigned'/'completed')` 传递参数（tab页面切换后无法通过URL传参，改用 storage 中转）。
2. **index页（`src/pages/index/index.tsx`）**：
   - 新增 `completed` tab（已完成）
   - `useDidShow` 回调读取 `pendingFilter`，自动 `setActiveTab` 并触发请求
   - 类型扩展为 `type FilterType = 'created' | 'assigned' | 'completed' | 'all'`
3. **后端（`server/src/tasks/tasks.service.ts` + `tasks.controller.ts`）**：`filter=completed` 时自动加 `assignee_id=userId AND status=done` 双重筛选。

**关键技术点**：微信小程序 tab 页面之间不能用 URL 参数传递（`switchTab` 的 url 不支持 query），改用 Storage 中转 → 在 `useDidShow` 里读取并清除。

## 修复：任务列表卡片空白（显示 ID 而非姓名）

**问题**：任务卡片负责人区域显示 `user-001` 等原始 ID，看起来像是空白/乱码。

**根本原因**：
1. 后端 `listTasks` 的 SELECT 只查了 `assignee_id`，没有 JOIN users 表
2. 前端直接用 `task.assignee_id` 渲染，而非 `task.assignee.name`

**修复**：
- 后端：`tasks.service.ts` 的 SELECT 改用 Supabase 外键关联语法：`assignee:users!tasks_assignee_id_fkey(id, name, avatar)` 和 `creator:users!tasks_creator_id_fkey(id, name, avatar)`
- 前端：Task interface 新增 `assignee` 和 `creator` 嵌套对象字段；卡片显示 `task.assignee?.name || task.assignee_id`；同时展示任务描述；修复 `gap-2` → `style={gap:'8px'}` 兼容性问题
